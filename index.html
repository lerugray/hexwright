        Hexwright Help

        Getting loaded

        • Use Load GotA sample for a ready-to-edit example project.
        • Open with ?project=<manifest-url> to boot directly into a manifest (for example ?project=samples/twu-project.json).
        • Or load your own files with the top bar pickers: Map, Grid, Terrain, and Sides.

        View and opacity controls

        • Map: scan + trace overlays.
        • Classification: terrain/features/hexsides only (no scan photo).
        • Both: scan and classification together; press V to cycle modes.
        • Overlay → Terrain fill opacity controls fill strength in Both view.
        • Traces → Opacity sets trace image transparency.

        Painting and alignment

        • Edge paint (E): pick a feature chip, then click an edge to toggle it, drag to paint it on, hold Alt to erase. One drag stroke becomes one undo step.
        • Brush (B): paint terrain onto hexes directly.
        • Nudge map (N): drag the scan under the grid, or use arrow keys; hold Shift with arrows for 10-pixel steps. The map offset autosaves with the project.

        Inspector, history, and save safety

        • Click a hex to open the inspector and edit terrain, in-hex features, and edge features.
        • Use Undo (button or Ctrl/Cmd + Z) to step back edits.
        • Redo history is tracked internally, but this build does not expose a redo button or shortcut.
        • Sessions autosave per project in your browser storage and can be restored when you reopen the app or load a matching manifest.

        Checks and data pipelines

        • Anomalies highlights unclassified hexes, orphan hexsides, and draft hexes.
        • Import WMP draft loads parser terrain output as draft so you can refine and confirm it.
        • Import TWU layer accepts one file at a time (rivers or rail).
        • Export → TWU rivers+rail downloads both rivers.json and rail.json.
